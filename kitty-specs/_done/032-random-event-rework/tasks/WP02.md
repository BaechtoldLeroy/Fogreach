# WP02 — New event types: shrine buffs, NPC rescue, cursed items

## Objective

Add three new choice-driven event types to the random event pool using the choice UI from WP01. These events offer meaningful risk/reward decisions that make dungeon runs more varied and interesting.

## Context

Depends on WP01's `showEventChoiceDialog()` and `registerEventType()`. Each new event uses the choice system to present the player with 2-3 options.

Key files for integration:
- `js/eventSystem.js` — register new events via `registerEventType()`
- `js/statusEffects.js` — apply shrine buffs and cursed item debuffs
- `js/lootSystem.js` — generate cursed item rewards

## Subtasks

### T001 — Shrine buff event

**Purpose**: Temporary stat boost with a tradeoff. Player chooses which shrine to pray at.

**Steps**:
1. Register `shrine_buff` event (weight: 10, minDepth: 2).
2. Handler returns a choice dialog with 2-3 shrine options, e.g.:
   - "Schrein der Staerke" — +25% damage, -15% armor for 60 seconds
   - "Schrein der Ausdauer" — +30% armor, -10% speed for 60 seconds
   - "Weitergehen" — decline, no effect
3. On selection, apply the buff+debuff via `statusEffects.js` with a timed duration.
4. Show a toast confirming the chosen buff.

**Files**: `js/eventSystem.js` (~50 lines), `js/statusEffects.js` (~20 lines for timed buff support if not present)

**Validation**: Triggering the shrine event shows a choice dialog. Selecting a shrine applies the correct buff and debuff. After 60 seconds both expire.

### T002 — NPC rescue event

**Purpose**: Player finds a trapped NPC and can choose to help for a reward or ignore.

**Steps**:
1. Register `npc_rescue` event (weight: 8, minDepth: 3).
2. Handler returns a choice dialog:
   - "Retten" — spend 10 seconds fighting spawned enemies, then receive gold + item reward
   - "Ignorieren" — no risk, no reward
3. If player chooses to rescue, spawn 3-5 enemies near the event location. After all are defeated, grant reward via `lootSystem.js`.
4. Show toast with outcome.

**Files**: `js/eventSystem.js` (~60 lines)

**Validation**: Choosing "Retten" spawns enemies. Defeating them grants gold and possibly an item. Choosing "Ignorieren" does nothing.

### T003 — Cursed item event

**Purpose**: High-power item with a debuff — classic risk/reward.

**Steps**:
1. Register `cursed_item` event (weight: 6, minDepth: 4).
2. Handler returns a choice dialog:
   - "Aufheben" — receive a high-tier item but apply a permanent-until-unequip debuff (e.g., -20% max HP)
   - "Zerstoeren" — destroy the item for bonus gold
   - "Liegenlassen" — walk away
3. Cursed items get a visual indicator (purple name text or "Verflucht" prefix).
4. Debuff is tracked via `statusEffects.js` and removed when the cursed item is unequipped.

**Files**: `js/eventSystem.js` (~50 lines), `js/statusEffects.js` (~15 lines), `js/lootSystem.js` (minor: flag for cursed items)

**Validation**: Taking a cursed item adds it to inventory with debuff active. Unequipping removes the debuff. Destroying yields gold.

## Definition of Done

- Three new event types registered and functional: `shrine_buff`, `npc_rescue`, `cursed_item`.
- Each uses WP01's choice dialog with appropriate options.
- Shrine buffs expire after 60 seconds.
- NPC rescue spawns enemies and grants reward on completion.
- Cursed items apply debuffs that track with equip state.
- All events have German-language UI text consistent with existing events.
- No console errors; existing events unaffected.

|---|---|
