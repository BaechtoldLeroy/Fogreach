# WP08 — Quest System Migration

## Branch Strategy

- Planning/base branch: `main`
- Stack base: `WP01` (use `--base WP01`)
- Merge target: `main`

## Objective

Migriere `questSystem.js` (~95 Strings — größtes Modul) komplett: Quest-Definitionen (Titel, Descriptions), NPC-Dialoge (Offer, Progress, Complete), Tracker-UI-Texte. Alle Quest-Strings via i18n-Keys, Quest-Definitionen halten Keys statt Literale.

## Context

- `questSystem.js` enthält `QUEST_DEFINITIONS`-Object mit pro Quest: `title`, `description`, `dialogueOffer`, `dialogueProgress`, `dialogueComplete`.
- ~5 Quests à ~3-5 Strings + Tracker-UI-Generika ≈ 95 Strings.
- Dialoge können mehrzeilig sein, ggf. Arrays von Strings.
- Quest-Tracker-UI lebt während Game (HUD-overlay rechts oben).
- WP01 garantiert `window.i18n`.

### Subtask T041 — Render-Site Lookup via Helper

**Steps**:
1. Helper anlegen (in `questSystem.js`):
   ```js
   function getQuestTitle(quest) { return window.i18n.t(quest.titleKey); }
   function getQuestDescription(quest) { return window.i18n.t(quest.descriptionKey); }
   function getQuestDialogue(quest, phase) {
     const key = quest['dialogue' + phase[0].toUpperCase() + phase.slice(1) + 'Key'];
     return window.i18n.t(key);
   }
   ```
2. Window-exposen falls externe Verbraucher (z.B. NPC-Dialog-Renderer in main.js):
   ```js
   window.QuestSystem = window.QuestSystem || {};
   window.QuestSystem.getQuestTitle = getQuestTitle;
   window.QuestSystem.getQuestDescription = getQuestDescription;
   window.QuestSystem.getQuestDialogue = getQuestDialogue;
   ```
3. Alle internen Calls auf `quest.title` → `getQuestTitle(quest)`.

**Files**: `js/questSystem.js` (modify)

## Risks

- **Größtes Modul, viele Stellen**: am höchsten Migration-Risk. → Mitigation: systematisch pro Quest-Block durchgehen, danach grep über deutsche Wörter (Quest-Begriffe wie "Quest", "Aufgabe", "abgeschlossen").
- **Save-Game-Kompatibilität**: alte Saves haben aufgelöste Quest-Titel im State? → Quest-State sollte nur `quest.id` speichern, Strings dynamisch auflösen. Falls doch Strings persistiert sind: Migration-Logik beim Laden.
- **Externe Caller**: NPC-Dialog-Renderer (vermutlich in `main.js` oder Hub-Scene) liest `quest.title` → muss auf `QuestSystem.getQuestTitle(quest)` umgestellt werden. ABER: das ist ein Cross-File-Call → out-of-scope für diesen WP. Mitigation: getter-Pattern auf Quest-Object: `Object.defineProperty(quest, 'title', { get() { return i18n.t(this.titleKey); }})` → externe Caller merken nichts.
