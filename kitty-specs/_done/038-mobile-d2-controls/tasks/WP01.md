# WP01 — Tap-to-move on mobile

## Objective

Implement tap-to-move and hold-to-move for mobile touch devices. A single tap on a walkable tile sets the player's movement target; holding down continuously updates the target position so the player walks toward the touch point.

## Context

Desktop already has D2-style click-to-move via the `movementWeight` setting in SettingsScene (0 = instant, 1 = D2-like weighted movement). Mobile currently uses a virtual joystick (rex plugin) for movement. This WP adds tap/hold movement alongside the joystick -- the joystick overrides tap-move when actively deflected.

## Subtasks

### T001 — Tap-to-move handler

**Purpose**: Detect single taps on walkable floor tiles and set the player movement target.

**Steps**:
1. In `js/mobileControls.js`, add a `pointerdown` listener that fires only on mobile (`isMobile` check).
2. Convert the tap screen coordinates to world coordinates using `scene.cameras.main.getWorldPoint()`.
3. Check that the tapped tile is walkable (use existing collision/tilemap lookup).
4. Set the player's movement target to the tapped world position, reusing the desktop D2 movement logic with `movementWeight = 1`.
5. Ensure taps on UI elements (HUD, ability buttons) are ignored by checking Phaser input priority / checking if the pointer is over any interactive UI game object.

**Files**: `js/mobileControls.js`, `js/player.js`.

**Validation**: Tap a walkable area on mobile -- the player walks to that position. Tap on HUD buttons -- no movement triggered.

### T002 — Hold-to-move (continuous movement)

**Purpose**: Holding a position makes the player walk continuously toward the touch point.

**Steps**:
1. Track `pointerdown` and `pointermove` events while the pointer is held down.
2. Continuously update the player's movement target to the current touch position (throttle to ~10 updates/sec to avoid performance issues).
3. On `pointerup`, stop updating the target (player finishes walking to last known position).
4. If the joystick is actively deflected (`force > deadzone`), ignore hold-to-move input.

**Files**: `js/mobileControls.js`.

**Validation**: Hold finger on screen and drag -- player follows the touch point smoothly. Release -- player stops at last target.

### T003 — Joystick coexistence

**Purpose**: Ensure tap/hold movement and the virtual joystick coexist without conflict.

**Steps**:
1. When the joystick force exceeds the dead zone threshold, disable tap/hold movement processing.
2. When the joystick is released (force drops below threshold), re-enable tap/hold movement.
3. The joystick zone (bottom-left area) should not trigger tap-to-move.

**Files**: `js/mobileControls.js`.

**Validation**: Using the joystick overrides any active tap-move. Releasing the joystick allows tap-to-move again. Tapping inside the joystick zone does not trigger walk-to.

## Definition of Done

- Tapping a walkable tile on mobile moves the player to that position.
- Holding and dragging moves the player continuously toward the touch point.
- The joystick overrides tap/hold movement when actively used.
- Tapping UI elements does not trigger movement.
- Desktop controls remain unaffected.

## Risks

- Pointer event conflicts between joystick plugin and new tap handlers. May need to filter by pointer ID.
- Walkable tile detection must match the collision layer used by the pathfinding/movement system.
