# WP04 — D2 controls settings toggle

## Objective

Add a "D2 Controls" toggle in the mobile section of SettingsScene. When enabled, tap-to-move and tap-to-attack are active alongside the joystick. When disabled, the classic joystick-only control scheme is used.

## Context

The SettingsScene already has a mobile section with toggles (e.g., auto-aim from spec 021). This WP adds a new toggle that gates all the behavior introduced in WP01-WP03. The setting should persist across sessions using the existing settings persistence mechanism (localStorage or similar).

## Subtasks

### T001 — Settings UI toggle

**Purpose**: Add the D2 Controls toggle to the SettingsScene mobile section.

**Steps**:
1. In `js/scenes/SettingsScene.js`, add a new toggle row in the mobile settings section labeled "D2 Controls" with a description like "Tap to move and attack (Diablo 2 style)".
2. Store the setting value in `window.__MOBILE_D2_CONTROLS__` (default: `true`).
3. Persist the setting using the same mechanism as other mobile settings (localStorage key, e.g., `demonfall_d2_controls`).
4. Load the persisted value on game startup.

**Files**: `js/scenes/SettingsScene.js`.

**Validation**: The toggle appears in the mobile settings section. Toggling it changes `window.__MOBILE_D2_CONTROLS__`. The value persists across page reloads.

### T002 — Gate tap/hold behavior on the setting

**Purpose**: Disable all D2-style touch controls when the toggle is off.

**Steps**:
1. In `js/mobileControls.js`, wrap all tap-to-move, hold-to-move, and tap-to-attack logic with a check for `window.__MOBILE_D2_CONTROLS__`.
2. When the setting is `false`, only the virtual joystick and existing ability buttons function.
3. When toggled at runtime (without scene restart), immediately enable/disable the tap handlers. If live-toggle is complex, require a scene restart and display a note in the settings UI.

**Files**: `js/mobileControls.js`.

**Validation**: With D2 Controls off, tapping the game world does nothing -- only joystick works. With D2 Controls on, tap-to-move and tap-to-attack function as designed. Toggling mid-game works (or shows restart notice).

## Definition of Done

- "D2 Controls" toggle appears in the mobile settings section.
- The toggle enables/disables all tap-to-move and tap-to-attack behavior.
- The setting persists across sessions.
- When disabled, the classic joystick-only mode works exactly as before.
- Desktop settings are not affected.

## Risks

- If live-toggle is not feasible, a scene restart requirement should be clearly communicated to the player.
- Default value (`true`) may surprise returning players -- consider showing a one-time tooltip or notification explaining the new control scheme.
