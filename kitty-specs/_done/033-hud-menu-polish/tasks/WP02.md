# WP02 — Consistent spacing, padding, and alignment

## Objective

Establish uniform spacing constants and apply them across all menu screens so that padding, margins, row heights, and element alignment are visually consistent.

## Context

After WP01 eliminates overlaps, each menu likely uses ad-hoc magic numbers for spacing. This WP standardizes those values so all menus share a consistent visual rhythm.

## Subtasks

### T001 — Define shared layout constants

Create a set of shared spacing/padding constants (e.g., `MENU_PADDING`, `ROW_HEIGHT`, `SECTION_GAP`, `HEADER_HEIGHT`) either in a shared module or as constants at the top of each scene file. Values should be derived from the 960x480 canvas size.

**Files**: All menu scene files.

### T002 — Apply constants to all menu scenes

Replace hardcoded offset values in inventory, shop, crafting, and settings with the shared constants. Ensure consistent left/right padding, row spacing, and header-to-content gaps.

**Files**: `js/inventory.js`, `js/scenes/ShopScene.js`, `js/scenes/CraftingScene.js`, `js/scenes/SettingsScene.js`

### T003 — Standardize text alignment and anchoring

Ensure all menu titles are centered, all labels are left-aligned with consistent indentation, and all value displays (gold, counts, stats) are right-aligned. Use consistent Phaser text `setOrigin()` calls.

**Files**: All menu scene files.

## Acceptance Criteria

- All menus use the same spacing constants for padding, row height, and section gaps.
- Headers, labels, and values are aligned consistently across all menu screens.
- Visual spacing feels uniform when switching between menus.
- No functional regressions.
