# Implementation Plan: Large Room Variety

**Branch**: `main` | **Date**: 2026-04-16 | **Spec**: `kitty-specs/030-large-room-variety/spec.md`

## Summary

Make large procedural rooms more interesting by adding decorative props, interactive objects, and cosmetic theme variations. Currently rooms are generated by BSP in `js/proceduralRooms.js` with floor/wall textures from `js/roomTemplates.js` but lack environmental detail.

## Technical Context

**Language/Version**: JavaScript (ES6, vanilla)
**Primary Dependencies**: Phaser 3.70.0
**Testing**: Manual playtesting
**Target Platform**: Desktop + Mobile browsers
**Performance Goals**: 60fps desktop, playable on mobile
**Constraints**: Procedural textures via Canvas 2D

## Approach

1. **Prop system** — Prop spawner places decorative objects (barrels, crates, pillars, torches, cobwebs, rubble) in large rooms after BSP generation. Collidable props join the obstacle group, non-collidable go on a decoration layer.

2. **Interactive objects** — Breakable barrels/crates drop minor loot via `js/lootSystem.js`. Hidden containers with small rewards.

3. **Cosmetic themes** — Extend room theme system in `js/roomTemplates.js` with palette/texture variations (overgrown, flooded, crumbling). Each theme defines floor tint, wall tint, and prop set.

4. **Placement** — Props placed on walkable tiles with clearance checks (no overlap with doors, stairs, spawn points).

## Key Files

- `js/proceduralRooms.js` — Add prop placement after room generation
- `js/graphics.js` — Add prop texture generation
- `js/roomTemplates.js` — Extend theme system with cosmetic variations
- `js/lootSystem.js` — Breakable object drops
