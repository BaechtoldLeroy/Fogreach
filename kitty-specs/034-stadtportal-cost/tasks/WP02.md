# WP02 — Portal Usage Check & UI Feedback

## Objective

Gate the Stadtportal button behind portal scroll availability. When the player has scrolls, using the portal consumes one. When none remain, the button is greyed out and a message is shown. Display the current scroll count near the portal button.

## Context

Read first:
- [../spec.md](../spec.md) — FR-004, FR-005
- [../plan.md](../plan.md) — Approach steps 3, 4
- WP01 must be completed first (provides `window.materialCounts.PORTAL_SCROLL`)

- `merge_target_branch`: `main`

## Subtasks

### T004 — Gate portal button behind scroll availability

**Purpose:** Prevent using the town portal without a scroll.

**Steps:**
1. Open `js/inventory.js`. Find the Stadtportal button handler (around line 667 per plan.md).
2. Before the teleport logic, add a check:
   ```js
   const scrollCount = (window.materialCounts && window.materialCounts.PORTAL_SCROLL) || 0;
   if (scrollCount <= 0) {
     // Show "no scrolls" feedback (see T005)
     return;
   }
   // Consume one scroll
   window.materialCounts.PORTAL_SCROLL = scrollCount - 1;
   ```
3. Ensure the scroll is consumed BEFORE the teleport executes, so it cannot be exploited by cancelling.
4. After consuming, trigger a HUD refresh if `window._refreshHUD` exists.

**Files:** `js/inventory.js` (~10 lines modified)

**Validation:** With 1 scroll, use portal — teleports and scroll count drops to 0. With 0 scrolls, press portal button — nothing happens (no teleport).

**Purpose:** Show the player how many scrolls they have and visually indicate when none are available.

**Steps:**
1. In `js/inventory.js`, find where the Stadtportal button is created/rendered.
2. Add a scroll count label near or on the button:
   ```js

   ```
3. When scroll count is 0, apply a greyed-out visual style to the button:

5. If the portal button also appears in the dungeon HUD (`js/main.js`), update that display as well.

## Definition of Done

- [ ] Portal button checks for scroll availability before teleporting
- [ ] Using the portal consumes exactly one scroll
- [ ] Button is greyed out when no scrolls are available
- [ ] Scroll count is displayed on or near the portal button
- [ ] "No scrolls" message appears when attempting to use with 0 scrolls
- [ ] Message auto-dismisses after 2-3 seconds

## Risks

| Risk | Mitigation |
|---|---|
| Portal button location moved since plan was written | Search for "Stadtportal" or "portal" in inventory.js to find current location |
| Scroll consumed but teleport fails | Consume scroll only at the point of no return, after all checks pass |
| No existing toast/notification system | Fall back to a simple Phaser text with a timed destroy |
| Button style not easily changed | Use alpha/tint changes which work on any Phaser element |
