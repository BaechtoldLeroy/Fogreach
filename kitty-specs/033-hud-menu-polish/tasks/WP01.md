# WP01 — Audit and fix overlapping UI elements

## Objective

Identify every instance of overlapping, clipped, or overflowing UI across the inventory panel, shop, crafting, and settings menus, then fix each by recalculating positions and sizes so no elements collide.

## Context

The plan identifies four menu surfaces with known overlap problems:
- **Inventory panel** (`js/inventory.js`) — equipment slots, grid items, and the Stadtportal button overlap at certain resolutions.
- **Shop scene** (`js/scenes/ShopScene.js`) — tab headers collide with the item list; gold display overlaps item entries.
- **Crafting scene** (`js/scenes/CraftingScene.js`) — material counts overlap recipe slot labels.
- **Settings scene** (`js/scenes/SettingsScene.js`) — sliders, toggles, and labels overlap when text is long.

## Subtasks

### T001 — Fix inventory panel overlaps

Audit `js/inventory.js` for overlapping equipment slots, grid items, and the Stadtportal button. Recalculate y-offsets and container heights so all elements have sufficient spacing and nothing overlaps at the default 960x480 resolution.

**Files**: `js/inventory.js`

### T002 — Fix shop scene overlaps

Fix tab header collision with item list area. Ensure the gold display does not overlap item entries. Verify scroll boundaries respect header/footer regions.

**Files**: `js/scenes/ShopScene.js`

### T003 — Fix crafting scene overlaps

Separate material count labels from recipe slot labels. Add vertical spacing between recipe rows so counts and slot graphics do not collide.

**Files**: `js/scenes/CraftingScene.js`

### T004 — Fix settings scene overlaps

Fix slider, toggle, and label overlaps. Ensure long label text does not run into control elements. Add consistent row height for each settings entry.

**Files**: `js/scenes/SettingsScene.js`

## Acceptance Criteria

- No UI elements overlap in any of the four menu screens at 960x480 resolution.
- Text does not overflow or clip its container in any menu.
- All existing menu functionality is preserved (buttons, scrolling, interactions).
