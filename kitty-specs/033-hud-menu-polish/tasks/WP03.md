# WP03 — Visual consistency — fonts, colors, and button styles

## Objective

Standardize font families, font sizes, text colors, background panel colors, and button/tab styles so all menus share a unified visual identity.

## Context

Different menus currently use slightly different font sizes, color values, and button rendering approaches. This WP brings them all into alignment using a consistent style palette. Tier colors (for item rarity) should remain distinct but be applied consistently.

## Subtasks

### T001 — Define a shared style palette

Establish canonical values for: heading font size, body font size, label color, value color, panel background color/alpha, button fill color, button hover color, and tier/rarity colors. Document these as constants.

**Files**: Shared constants or top of each scene file.

### T002 — Apply font and color consistency

Update all `this.add.text()` calls across menus to use the canonical font sizes and colors. Ensure headings, body text, and labels use the same style config objects.

**Files**: `js/inventory.js`, `js/scenes/ShopScene.js`, `js/scenes/CraftingScene.js`, `js/scenes/SettingsScene.js`

### T003 — Standardize button and tab rendering

Ensure all interactive buttons (close, buy, craft, navigate tabs) use the same rounded-rect style, fill color, hover effect, and text style. Replace any one-off button rendering with a shared helper or consistent inline pattern.

**Files**: All menu scene files, potentially `js/main.js` for HUD buttons.

## Acceptance Criteria

- All menus use the same font sizes for equivalent text roles (headings, labels, values).
- Color palette is consistent: same background tints, same text colors, same button styles.
- Tier/rarity colors are applied consistently wherever item rarity is shown.
- No visual regressions in the HUD or gameplay elements.

## Key Files
