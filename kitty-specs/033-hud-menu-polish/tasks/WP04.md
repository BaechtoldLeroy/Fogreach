# WP04 — Responsive layout for desktop and mobile

## Objective

Ensure all menus render correctly on both desktop (960x480) and mobile (portrait and landscape) by using canvas-relative positioning and safe-area insets rather than hardcoded pixel values.

## Context

The game uses Phaser's FIT scale mode on a 960x480 canvas. Menus need to adapt to different aspect ratios without overlaps or clipping. The safe-area module from spec 021 (`js/mobileSafeArea.js`) provides inset values that menus should respect.

## Subtasks

### T001 — Canvas-relative positioning for menu panels

Replace any absolute pixel positions with values derived from `this.cameras.main.width` and `this.cameras.main.height`. Menu panels should center correctly regardless of actual canvas dimensions.

**Files**: `js/inventory.js`, `js/scenes/ShopScene.js`, `js/scenes/CraftingScene.js`, `js/scenes/SettingsScene.js`

### T002 — Safe-area respect in menu overlays

Ensure menu panels do not extend into safe-area inset regions on mobile. Read insets from `window.__SAFE_AREA__` and offset panel edges accordingly. On desktop (insets = 0), behavior is unchanged.

**Files**: All menu scene files.

### T003 — Test and fix at multiple aspect ratios

Verify menus at 960x480 (desktop), 390x844 portrait (iPhone), 844x390 landscape (iPhone), and 360x800 (Android). Fix any overflow, clipping, or overlap found at these resolutions.

**Files**: All menu scene files, potentially `js/mobileControls.js`.

## Acceptance Criteria

- All menus display correctly at 960x480 desktop resolution.
- All menus display correctly on mobile portrait and landscape emulations without overlaps or clipping.
- Safe-area insets are respected on notched devices.
- No functional regressions on either platform.
