# WP03 — Touch priority and UI conflict resolution

## Objective

Ensure that touch input on mobile UI elements (HUD, ability buttons, menus, joystick) is never intercepted by the tap-to-move or tap-to-attack systems. Establish a clear input priority hierarchy.

## Context

WP01 and WP02 add new pointer listeners for game-world interaction. These must not fire when the player is tapping ability buttons, opening menus, or using the joystick. Phaser's input priority system and `event.stopPropagation()` patterns need to be configured correctly.

## Subtasks

### T001 — Input priority hierarchy

**Purpose**: Define and enforce a clear priority order for touch events.

**Steps**:
1. Set Phaser input depth/priority on all interactive UI elements (ability buttons, HUD icons, menu buttons) so they consume pointer events before the game-world tap handler.
2. In the tap-to-move/attack handler, check `pointer.wasTouch` and verify no UI element was hit. Use `scene.input.hitTestPointer(pointer)` to detect if an interactive object is under the touch point.
3. If any UI element is under the pointer, skip all game-world tap processing.
4. Define explicit zones: joystick area (bottom-left), ability buttons (bottom-right), HUD elements (top) -- taps in these regions never trigger world interactions.

**Files**: `js/mobileControls.js`.

**Validation**: Tapping ability buttons fires abilities without triggering movement. Tapping the HUD opens menus without walking. No "phantom walks" when interacting with UI.

### T002 — Edge case handling

**Purpose**: Handle tricky multi-touch and transition scenarios.

**Steps**:
1. When the player opens a full-screen menu (inventory, settings), disable all game-world tap processing until the menu closes.
2. Handle multi-touch correctly: if one finger is on the joystick and another taps an enemy, only the enemy-tap should process (do not interfere with the joystick finger).
3. On scene transitions, clean up all tap/hold state to avoid stale targets or phantom movement in the new scene.

**Files**: `js/mobileControls.js`.

**Validation**: Opening the inventory menu and tapping inside it does not cause the player to move. Multi-touch with joystick + tap works correctly. Scene transitions do not cause input bugs.

## Definition of Done

- UI elements always take priority over game-world taps.
- No phantom movement when interacting with HUD, menus, or ability buttons.
- Multi-touch scenarios work correctly (joystick + tap).
- Scene transitions clean up tap/hold state properly.
- Desktop controls remain unaffected.

## Risks

- Phaser's input priority system may behave differently across mobile browsers. Test on iOS Safari and Chrome Android.
- Multi-touch pointer ID tracking can be error-prone.
