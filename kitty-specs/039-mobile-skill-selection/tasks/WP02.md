# WP02 — Cooldown indicators on mobile ability buttons

## Objective

Add visual cooldown feedback on mobile ability buttons so players can see when each skill becomes available again. Currently cooldowns are tracked internally in `abilitySystem.js` but have no visual representation on mobile.

## Context

- `abilitySystem.js` tracks cooldown timers per ability (`cooldownRemaining`, `cooldownTotal`).
- Mobile ability buttons are rendered as Phaser circles/graphics in `mobileAbilityButtons.js`.
- Desktop shows cooldown info in the HUD but mobile has no equivalent visual.

## Subtasks

### T001 — Radial cooldown overlay

**Purpose**: Show a radial sweep or darkening fill over the ability button while on cooldown.

**Steps**:
1. For each ability button, add a Phaser Graphics overlay that draws an arc proportional to `cooldownRemaining / cooldownTotal`.
2. Update the arc every frame in the scene's `update()` loop (or via a timed event).
3. Use semi-transparent dark fill (alpha 0.6) to clearly show the button is unavailable.
4. When cooldown reaches 0, remove the overlay and optionally flash the button briefly.

**Files**: `js/mobileAbilityButtons.js`.

**Validation**: After using an ability with a cooldown, the button visually fills back up over the cooldown duration.

### T002 — Cooldown text timer

**Purpose**: Show remaining seconds as text on the button for precise feedback.

**Steps**:
1. Add a small text element centered on the button showing remaining seconds (e.g., "3.2").
2. Update text each frame from `abilitySystem.js` cooldown state.
3. Hide text when cooldown is 0.
4. Use a readable font size (14-16px) with outline/shadow for visibility.

**Files**: `js/mobileAbilityButtons.js`.

**Validation**: Cooldown number counts down visually and disappears when ready.

### T003 — Prevent activation during cooldown

**Purpose**: Provide tactile feedback when tapping a skill on cooldown.

**Steps**:
1. When a player taps a button that is on cooldown, show a brief shake animation or red flash instead of activating.
2. Optionally dispatch `demonfall:ability-cooldown-blocked` event for haptics (if haptics WP is merged).

**Files**: `js/mobileAbilityButtons.js`, `js/mobileControls.js`.

**Validation**: Tapping a cooling-down button does not fire the ability and shows visual feedback.

## Definition of Done

- Every mobile ability button shows a radial cooldown indicator when the skill is on cooldown.
- Remaining cooldown seconds are displayed as text on the button.
- Tapping a button on cooldown provides visual feedback instead of silently failing.
- Cooldown visuals clear immediately when the skill becomes available.
- No performance regression (overlay updates are efficient).

## Risks

- Per-frame Graphics redraws for 6 buttons could impact performance on low-end mobile. Consider using a pre-rendered sprite sheet or updating only when the value changes significantly (e.g., every 100ms).
