# WP01 — Skill selection trigger + overlay UI

## Objective

Add a way for mobile players to open a skill selection overlay and swap equipped abilities during gameplay. This is the core missing feature: currently there is no mobile equivalent of the desktop loadout screen (K key).

## Context

- Desktop players press K to open the loadout/inventory screen and swap skills. Mobile has no equivalent.
- `js/mobileControls.js` renders the ability button cluster (2x3 grid, bottom-right).
- `js/abilitySystem.js` tracks available skills, equipped state, and cooldowns.
- `js/inventory.js` contains the desktop loadout UI (reference for skill swap logic).

## Subtasks

### T001 — Add skill-swap trigger button or long-press gesture

**Purpose**: Give mobile players a way to enter skill selection mode.

**Steps**:
1. Add a small "swap" button (gear/swap icon) near the ability button cluster in `mobileControls.js`.
2. Alternatively or additionally, support long-press (500ms) on any ability slot to open selection for that slot.
3. Dispatch `demonfall:skill-select-open` event with `{slotIndex}` detail when triggered.

**Files**: `js/mobileControls.js`.

**Validation**: On mobile emulation, long-pressing an ability slot or tapping the swap button dispatches the event.

### T002 — Build skill selection overlay panel

**Purpose**: Display all available skills in a touch-friendly modal overlay.

**Steps**:
1. Create a centered overlay using Phaser graphics or DOM overlay showing all skills from `abilitySystem.js`.
2. Each skill shown as a tappable card/button with skill name and icon (minimum 44x44px touch target).
3. Highlight the currently equipped skill for the selected slot.
4. Overlay dims the background to indicate modal state.
5. Close overlay on tap-outside or a close button.

**Files**: `js/mobileAbilityButtons.js`, `js/mobileControls.js`.

**Validation**: Overlay appears centered, shows all available skills, and dismisses correctly.

### T003 — Wire skill swap logic

**Purpose**: When a player selects a skill from the overlay, equip it in the chosen slot.

**Steps**:
1. On skill tap in the overlay, call into `abilitySystem.js` to update the equipped skill for that slot.
2. Update the mobile ability button to reflect the new skill.
3. Dispatch `demonfall:skill-equipped` event with `{slotIndex, skillKey}`.
4. Close the overlay after selection.

**Files**: `js/mobileControls.js`, `js/abilitySystem.js`.

**Validation**: Swapping a skill updates the button and the ability fires correctly on next tap.

### T004 — Active skill highlighting on the skill bar

**Purpose**: Visually indicate which skill is currently active/selected.

**Steps**:
1. Add a highlight border or glow effect to the active ability button when it is in use (casting, channeling).
2. Use the existing ability state from `abilitySystem.js` to drive the highlight.

**Files**: `js/mobileAbilityButtons.js`.

**Validation**: When an ability is activated, its button shows a visible highlight that clears when the ability ends.

## Definition of Done

- Mobile players can open a skill selection overlay via long-press or dedicated button.
- Overlay shows all available skills with touch-friendly targets (>= 44x44px).
- Selecting a skill equips it in the chosen slot and updates the button.
- Active skill is visually highlighted on the skill bar.
- Overlay does not obstruct gameplay when closed.
- No regression on desktop behavior.

## Risks

- Long-press gesture may conflict with existing ability hold behavior (e.g., charge slash). Need to differentiate hold-to-cast (shorter) from hold-to-swap (longer, 500ms+).
- Overlay rendering approach (Phaser vs DOM) affects z-index and input handling. DOM overlay is simpler for modal behavior.
