### Subtask T010 — Wechsel-Callback (Hot-Switch + Persist)

**Purpose**: Klick auf Sprachoption schaltet sofort um und persistiert.

**Steps**:
1. Im Click-Handler der Picker-Optionen:
   ```js
   onLanguageClick(newLang) {
     if (newLang === window.i18n.getLanguage()) return;
     window.i18n.setLanguage(newLang);
     window.persistence.setLanguage(newLang);
     // Re-Render: einfachster Weg ist Scene-Restart
     this.scene.restart();
   }
   ```
2. **Reihenfolge wichtig**: erst `i18n.setLanguage()` (triggert Subscriber, T011 hängt sich da rein), dann `persistence.setLanguage()` (persistiert), dann Restart.
3. `scene.restart()` ist OK weil Settings keine flüchtige Game-State hat. Alternative: alle UI-Elemente destroyen + neu rendern.

**Files**: `js/scenes/SettingsScene.js` (modify)

**Validation**: Klick auf "English" → Settings sofort komplett englisch, localStorage zeigt `language: 'en'`. Reload → bleibt englisch.

### Subtask T011 — `i18n.onChange()`-Subscriber

**Purpose**: Wenn extern (z.B. Debug-Tool) `i18n.setLanguage()` gerufen wird während Settings offen, soll Scene auch re-rendern.

**Steps**:
1. In `create()` nach Aufbau:
   ```js
   this._unsubscribeI18n = window.i18n.onChange(() => {
     // Re-Render ohne erneuten persistence-Write (sonst Loop)
     if (this.scene.isActive()) this.scene.restart();
   });
   ```
2. In `shutdown()` (oder analog Lifecycle-Hook): `if (this._unsubscribeI18n) this._unsubscribeI18n();`
3. **Idempotenz**: T010 ruft `i18n.setLanguage()` was Subscriber feuert was `restart()` triggert — restart wirft Scene weg, Subscriber-Cleanup MUSS in `shutdown` passieren, sonst Memory-Leak.

**Files**: `js/scenes/SettingsScene.js` (modify)

**Validation**: Settings öffnen → in Konsole `i18n.setLanguage('en')` → Settings re-rendert sofort englisch. `shutdown` wird aufgerufen (z.B. mit Console-Log temporär verifizieren).

## Risks

- **Endless-Loop bei Subscriber**: wenn Subscriber `setLanguage()` aufruft, infinite recursion. → Mitigation: Subscriber NUR re-rendert, nie `setLanguage` aufruft. T010 ruft `setLanguage` direkt — Subscriber sieht das nur als Notification.
- **Restart verliert State**: Wenn Settings interne State-Werte (z.B. unsaved Changes) hat, geht das beim restart verloren. → Bei diesem Game speichert jede Toggle-Änderung sofort in `saveSettings()`, also unproblematisch.
- **String-Vollständigkeit**: leicht Strings übersehen. → Mitigation: nach Migration `grep` über die Datei, alle nicht-Key-Strings in UI-Code prüfen.
