# WP03 — Start Scene + Loading Screen Migration

## Branch Strategy

- Planning/base branch: `main`
- Stack base: `WP01` (use `--base WP01`)
- Merge target: `main`

## Objective

Migriere `startScene.js` (~5 Strings: Loading-Texte, Title-Banner, Start-Button) komplett auf i18n-Keys.

## Context

- WP01 garantiert `window.i18n` mit Bootstrap-Sprache aktiv.
- Start-Scene läuft VOR allen anderen Scenes — beim ersten Render ist die persistierte Sprache bereits gesetzt.

## Implementation

### Subtask T012 — DE/EN Strings registrieren (Modulkopf)

**Steps**:
1. Am Modulkopf von `js/scenes/startScene.js`:
   ```js
   if (window.i18n) {
     window.i18n.register('de', {
       'start.loading': 'Lade...',
       'start.loading_assets': 'Lade Assets...',
       'start.title': 'DEMONFALL',
       'start.press_to_start': 'Klicken zum Starten',
       'start.continue': 'Fortsetzen',
       // ...alle gefundenen Strings
     });
     window.i18n.register('en', {
       'start.loading': 'Loading...',
       'start.loading_assets': 'Loading assets...',
       'start.title': 'DEMONFALL',
       'start.press_to_start': 'Click to start',
       'start.continue': 'Continue',
     });
   }
   ```
2. Vollständigkeit per Code-Inspektion: alle `.add.text(...)` und `setText(...)` Calls mit String-Literalen enumerieren.

**Files**: `js/scenes/startScene.js` (modify)

---

### Subtask T013 — Hardcoded Strings auf `i18n.t()`

**Steps**:
1. Jede `.add.text(x, y, 'Lade...')` → `.add.text(x, y, window.i18n.t('start.loading'))`.
2. `setText('...')`-Calls analog auf `setText(window.i18n.t('start.xxx'))`.
3. Strings die _während_ der Scene aktualisiert werden (z.B. Loading-Progress-Text): in der Update-Funktion `i18n.t()` aufrufen, NICHT in einer Modul-Konstante zwischenspeichern.

**Files**: `js/scenes/startScene.js` (modify)

---

### Subtask T014 — Loading-Progress-Texte

**Steps**:
1. Falls Loading-Progress dynamische Werte hat (z.B. `'Lade Assets... 47%'`):
   - Key mit Platzhalter: `'start.loading_progress': 'Lade Assets... {percent}%'`
   - Aufruf: `i18n.t('start.loading_progress', { percent: 47 })`
2. Phaser-LoaderEvents (`'progress'`, `'complete'`) ggf. anhaken und Text aktualisieren.

**Files**: `js/scenes/startScene.js` (modify)

---

### Subtask T015 — Manueller Test in beiden Sprachen

**Steps**:
1. localStorage `language: 'de'` → Reload → Start-Scene komplett deutsch
2. In Konsole: `persistence.setLanguage('en')` + Reload → Start-Scene komplett englisch
3. Console: keine `[MISSING:start.*]`-Warnings
4. `localStorage.clear()` + Reload → Default deutsch

**Files**: keine — nur Verifikation

## Risks

- **Sprachwechsel _während_ Loading**: Loading dauert kurz, kein realistisches Szenario. Falls doch: `i18n.onChange()` Subscriber NICHT nötig — Scene wird sowieso schnell verlassen.
- **Strings die in Konstanten zwischengespeichert werden**: würde Bootstrap-Sprache einfrieren. → Mitigation: immer `t()` zur Render-Zeit aufrufen.

## Reviewer Guidance

- Beide Sprachen einmal sehen (mit `localStorage.setItem('demonfall_settings_v1', JSON.stringify({language:'en'}))` + Reload).
- Keine deutschen Literale außerhalb `register('de',...)` Block.
