# WP05 — Loadout Overlay Migration

## Branch Strategy

- Planning/base branch: `main`
- Stack base: `WP01` (use `--base WP01`)
- Merge target: `main`

## Objective

Migriere `loadoutOverlay.js` (~13 Strings: UI-Hints, Slot-Labels, Bestätigungs-Buttons) auf i18n-Keys. Subscriber für Hot-Switch falls Overlay offen ist.

## Context

- Loadout-Overlay öffnet sich via `L`-Taste oder Menü.
- Zeigt verfügbare Abilities + aktuell zugewiesene Slots.
- WP01 garantiert `window.i18n`.

## Implementation

### Subtask T023 — DE/EN Strings registrieren

**Steps**:
1. Modulkopf von `js/loadoutOverlay.js`:
   ```js
   if (window.i18n) {
     window.i18n.register('de', {
       'loadout.title': 'Fähigkeiten',
       'loadout.hint': 'Klicke auf einen Slot, dann auf eine Fähigkeit',
       'loadout.slot.attack': 'Angriff',
       'loadout.slot.ability_1': 'Slot 1',
       'loadout.slot.ability_2': 'Slot 2',
       'loadout.slot.ability_3': 'Slot 3',
       'loadout.confirm': 'Bestätigen',
       'loadout.cancel': 'Abbrechen',
       'loadout.locked': 'Gesperrt',
       'loadout.unlocked_at_level': 'Schalt frei in Level {level}',
       // ...alle weiteren strings (~13 erwartet)
     });
     window.i18n.register('en', {
       'loadout.title': 'Abilities',
       'loadout.hint': 'Click a slot, then an ability',
       'loadout.slot.attack': 'Attack',
       'loadout.slot.ability_1': 'Slot 1',
       'loadout.slot.ability_2': 'Slot 2',

       'loadout.unlocked_at_level': 'Unlocks at level {level}',
     });
   }
   ```
2. Abilities werden vermutlich aus einer separaten Quelle gelesen (skillSystem oder abilitySystem) — deren Namen migrieren wir hier NICHT (out-of-scope, würde Ownership-Konflikt geben). Falls Ability-Namen direkt in `loadoutOverlay.js` hardcoded sind, gehören sie hierher.

**Files**: `js/loadoutOverlay.js` (modify, top-of-file)

### Subtask T024 — Hardcoded Strings auf `i18n.t()`

**Files**: `js/loadoutOverlay.js` (modify)

### Subtask T025 — `onChange`-Subscriber + Cleanup

**Steps**:
1. Beim Overlay-Öffnen Subscriber registrieren:
   ```js
   if (window.i18n && !this._unsubscribeI18n) {
     this._unsubscribeI18n = window.i18n.onChange(() => {
       if (this.isOpen) this.refreshTexts(); // oder rebuildOverlay()
     });
   }
   ```
2. Beim Overlay-Schließen oder destroy: `if (this._unsubscribeI18n) { this._unsubscribeI18n(); this._unsubscribeI18n = null; }`
3. `refreshTexts()`: setze alle `Text`-Objekte mit `setText(i18n.t('loadout.xxx'))` neu.
4. **Falls Overlay komplett zerstört + neu gebaut wird beim Öffnen**: Subscriber kann auf Page-Lifetime registriert sein (kein Cleanup nötig), Re-Render nur wenn `isOpen`.

**Files**: `js/loadoutOverlay.js` (modify)
