# WP07 — Event System Migration

## Branch Strategy

- Planning/base branch: `main`
- Stack base: `WP01` (use `--base WP01`)
- Merge target: `main`

## Objective

Migriere `eventSystem.js` (~14 sichtbare Strings, evtl. mehr versteckt) komplett auf i18n-Keys. Inklusive Schrein, Glücksspiel, Verfluchte Truhe, Treasure-Hint und alle Toast-/Dialog-Texte.

## Context

- `eventSystem.js` enthält `EVENT_TYPES`-Array mit Event-Definitionen (`id`, `name`, `weight`, `minDepth`, `handler`).
- Handler spawnen Event-Objekte und zeigen Toasts/Choice-Dialogs via `showEventToast` und `showEventChoiceDialog`.
- Dialog-Choices haben `label` + `callback` — labels sind UI-Strings.
- WP01 garantiert `window.i18n`.
- **WICHTIG**: Der Shrine-Buff-Bug-Fix (siehe `inventory.js` recalcDerived + `eventBuffs`-Registry) ist BEREITS auf main gemerged. Dieser WP berührt nur die Strings, nicht die Buff-Logik.

### Subtask T033 — Event-Definitionen `name`-Field auf Key

**Steps**:
1. In jedem `EVENT_TYPES.push({...})`: `name: 'Mystischer Schrein'` → `nameKey: 'event.shrine.title'`.
2. Wo Events nach `name` sucht/anzeigt: `i18n.t(event.nameKey)`.
3. Falls die internen Event-IDs (`id: 'shrine_buff'`) als Key-Source dienen, kann auch der Key generisch konstruiert werden: `i18n.t('event.' + event.id + '.title')` — aber explizite Keys via `nameKey` sind robuster.
