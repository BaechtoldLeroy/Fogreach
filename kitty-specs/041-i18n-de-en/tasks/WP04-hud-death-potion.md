# WP04 — HUD + Death Screen + Potion (main.js)

## Branch Strategy

- Planning/base branch: `main`
- Stack base: `WP01` (use `--base WP01`)
- Merge target: `main`

## Objective

Migriere alle UI-Strings in `main.js` (~27 Strings: HUD-Labels, Death-Screen, Potion-Status, Stats-Display) auf i18n-Keys. Registriere `onChange`-Subscriber für HUD-Re-Render bei Sprachwechsel zur Laufzeit.

## Context

- `js/main.js` ist die größte Code-Datei der App; **modifiziere AUSSCHLIEßLICH UI-Text-Pfade**, keine Game-Logic.
- HUD wird via `updateHUD()` Funktion gerendert — Strings dort müssen `i18n.t()` zur Render-Zeit aufrufen, NICHT bei Modul-Load.
- Death-Screen wird einmalig beim Tod gespawnt — bei Sprachwechsel danach würde alter Text bleiben (akzeptabel; Sprachwechsel mitten im Death-Screen ist Edge-Case).
- WP01 garantiert `window.i18n`.

### Subtask T017 — `updateHUD()` auf `i18n.t()` umstellen

**Steps**:
1. `updateHUD()`-Funktion lokalisieren.
2. Jede `setText('...')` mit String-Konkatenation auf `i18n.t()`-Lookup umstellen:
   - Vorher: `healthText.setText('Leben: ' + playerHealth + '/' + playerMaxHealth)`
   - Nachher: `healthText.setText(i18n.t('hud.health') + ': ' + playerHealth + '/' + playerMaxHealth)` (oder Key mit Platzhaltern: `'hud.health.formatted': '{label}: {cur}/{max}'`)
3. Wichtig: `i18n.t()` IM `updateHUD()`-Body, nicht in einer Konstante außerhalb.

**Files**: `js/main.js` (modify)

**Validation**: Im Run sichtbar — HUD zeigt aktuelle Werte. Sprachwechsel → nach nächstem `updateHUD()`-Tick (i.d.R. <100ms) sieht man neue Sprache.

### Subtask T020 — `onChange`-Subscriber für HUD

**Steps**:
1. Nach Setup-Phase von `main.js` (z.B. nach Phaser-Game-Init oder in einer Scene-`create`):
   ```js
   if (window.i18n) {
     window.i18n.onChange(() => {
       if (typeof updateHUD === 'function') updateHUD();
       // optional: weitere lebende UI-Elemente neu rendern
     });
   }
   ```
2. Da `main.js` keine eigene Scene mit Lifecycle ist, KEIN Cleanup nötig (Subscriber lebt für die gesamte Session).

**Files**: `js/main.js` (modify)
