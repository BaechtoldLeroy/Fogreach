# WP01 — Fullscreen toggle in settings menu + persistence

## Objective

Add a fullscreen toggle control to the settings menu, wire it to the Phaser Scale Manager fullscreen API, save the preference to localStorage, and auto-restore it on game load (respecting browser user-gesture requirements).

## Context

Phaser 3.70.0 provides `this.scale.startFullscreen()` and `this.scale.stopFullscreen()` out of the box. There is commented-out fullscreen code in `js/main.js` (lines ~801-805) that can be referenced or uncommented. The settings are persisted under the `demonfall_settings_v1` localStorage key. The `SettingsScene.js` already has other toggles (sound, music, etc.) so the fullscreen toggle follows the same pattern.

## Subtasks

### T001 — Add fullscreen toggle to SettingsScene

**Purpose**: Give players a visible control to enter/exit fullscreen.

**Steps**:
1. In `js/scenes/SettingsScene.js`, add a checkbox/toggle row for "Fullscreen" alongside the existing settings controls.
2. On toggle ON: call `this.scale.startFullscreen()`.
3. On toggle OFF: call `this.scale.stopFullscreen()`.
4. Reflect the current fullscreen state when opening settings (listen to `fullscreenchange` event).

**Files**: `js/scenes/SettingsScene.js`.

**Validation**: Opening settings shows a fullscreen toggle. Toggling it enters/exits fullscreen. Re-opening settings shows the correct current state.

### T002 — Persist fullscreen preference in localStorage

**Purpose**: Remember the player's fullscreen preference across sessions.

**Steps**:
1. When the fullscreen toggle is changed, save the boolean under the `demonfall_settings_v1` localStorage key (alongside existing settings).
2. On game load, read the saved preference.
3. If fullscreen is preferred, queue a fullscreen request to fire on the first user interaction (click/tap), since browsers require a user gesture for the Fullscreen API.

**Files**: `js/scenes/SettingsScene.js`, `js/main.js`.

**Validation**: Toggle fullscreen on, refresh the page, click anywhere — game enters fullscreen automatically. Toggle off, refresh — game stays windowed.

### T003 — Verify canvas scaling in fullscreen

**Purpose**: Ensure the game canvas and HUD scale correctly when entering/exiting fullscreen.

**Steps**:
1. Confirm Phaser's `Scale.FIT` mode handles the resolution change.
2. Test that HUD elements (health, abilities, inventory) remain properly positioned in fullscreen.
3. Fix any positioning issues if found.

**Files**: `js/main.js`, `js/scenes/SettingsScene.js`.

**Validation**: Enter fullscreen — canvas fills the screen, HUD elements are properly positioned. Exit fullscreen — canvas returns to original size, no layout glitches.

## Acceptance Criteria

- Settings menu contains a working fullscreen toggle.
- Toggling fullscreen calls the correct Phaser Scale Manager methods.
- Fullscreen preference is saved to `demonfall_settings_v1` in localStorage.
- On reload, saved preference is restored on first user gesture.
- Canvas and HUD scale correctly in both windowed and fullscreen modes.
- No regression in existing settings functionality.

## Key Files

- `js/scenes/SettingsScene.js` — Fullscreen toggle UI and logic
- `js/main.js` — Existing commented-out fullscreen code, auto-restore on load
- `js/persistence.js` — localStorage helpers (if used)
