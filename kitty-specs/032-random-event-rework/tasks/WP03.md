# WP03 — New event types: elite ambush, puzzle room, gambling

## Objective

Add three more event types to complete the expanded event pool. These events focus on combat challenge (elite ambush), exploration (puzzle room), and economic risk (gambling).

## Context

Depends on WP01's choice UI and `registerEventType()`. Can be implemented in parallel with WP02 since both register independent event types.

Key files:
- `js/eventSystem.js` — register new events
- `js/eliteEnemies.js` — spawn elite enemy for ambush event
- `js/lootSystem.js` — reward generation

## Subtasks

### T001 — Elite ambush event

**Purpose**: Mini-boss encounter with better loot as reward.

**Steps**:
1. Register `elite_ambush` event (weight: 8, minDepth: 5).
2. Handler shows a warning toast ("Ein maechiger Feind naehert sich!") then spawns an elite enemy from `eliteEnemies.js`.
3. Elite enemy has 2x HP and 1.5x damage compared to floor-appropriate enemies.
4. On defeat, drop a guaranteed rare-tier item via `lootSystem.js` plus bonus gold.
5. No choice dialog needed — this is a forced combat encounter (fire-and-forget handler).

**Files**: `js/eventSystem.js` (~40 lines), `js/eliteEnemies.js` (~30 lines for elite spawn helper if not present)

**Validation**: Event spawns a visibly tougher enemy. Defeating it drops better loot than normal enemies.

### T002 — Puzzle room event

**Purpose**: Simple interactive puzzle that rewards exploration-minded players.

**Steps**:
1. Register `puzzle_room` event (weight: 7, minDepth: 3).
2. Handler presents a choice dialog with a simple pattern puzzle:
   - Display a sequence prompt (e.g., "Die Symbole zeigen: Feuer, Wasser, ???")
   - 3 answer choices, one correct
3. Correct answer: grant bonus XP + a random item.
4. Wrong answer: trigger a small trap (minor damage) but still grant partial XP.
5. Puzzle content drawn from a pool of 8-10 pre-written riddles to avoid repetition.

**Files**: `js/eventSystem.js` (~70 lines)

**Validation**: Puzzle dialog appears with three options. Correct choice grants full reward. Wrong choice deals minor damage and partial reward. Puzzles vary across encounters.

### T003 — Gambling event

**Purpose**: Risk gold for a chance at rare items.

**Steps**:
1. Register `gambling` event (weight: 6, minDepth: 2).
2. Handler shows a choice dialog:
   - "Kleiner Einsatz (50 Gold)" — low risk, common-to-uncommon item
   - "Grosser Einsatz (200 Gold)" — high risk, chance at rare item or lose gold
   - "Kein Interesse" — decline
3. Outcome determined by weighted random roll:
   - Small bet: 60% win (item worth ~100g), 40% lose
   - Big bet: 40% win (rare item worth ~500g), 60% lose
4. If player lacks sufficient gold, that option is grayed out / shows "(nicht genug Gold)".
5. Show result toast with win/lose feedback.

**Files**: `js/eventSystem.js` (~60 lines)

**Validation**: Gambling with sufficient gold works. Insufficient gold prevents selection. Win/lose outcomes are observable over multiple attempts. Gold is correctly deducted/awarded.

## Definition of Done

- Three new event types registered and functional: `elite_ambush`, `puzzle_room`, `gambling`.
- Elite ambush spawns a tougher enemy with better loot drops.
- Puzzle room presents varied riddles with correct/incorrect outcomes.
- Gambling correctly checks gold balance and applies win/lose outcomes.
- All events have German-language UI text.
- No console errors; no regressions in existing events.

## Risks

| Risk | Mitigation |
|---|---|
| eliteEnemies.js may not exist or have the right API | Check file exists; add spawn helper if needed |
| Puzzle riddles may feel repetitive with small pool | Start with 8-10 riddles; expand later |
| Gambling gold check race condition | Deduct gold immediately on selection, refund on win |
